POST /testsuiteruns

Tags: Test Suite Run

Create a new test suite run

A new test suite run, including the information about its test suite

name: string

The name of the test suite that was being run.

description: string

The description of the test suite

tags: object[]

The set of tags that indicates how to group this test suite

Tags
result: string

The result of the test suite if state was Finished, if this is specified the State is assumed to be Finished

startedAt: string (datetime)

The time when this run transitioned into Running. If not specified the server will use its the time of receiving this request. Only valid if result is specified.

endedAt: string (datetime)

The time when this run transitioned into the Finished state. Only valid if result is specified.

metadata: object

The metadata of this TestSuiteRun, any patched values end up here. This is not versioned so you can never assume this contains particular fields.

Uses default content-types: application/json

200 OK

The test suite run

Example for application/json
{
    "id": 1,
    "idStr": "1"
}

Schema definitions

Error: object

The response returned when a error happens

statusCode: integer (int32)

The HTTP status code of the error (also returned in the response)

message: string

Error message of what went wrong

fields: string

The fields that errored. Not always populated.

stackTrace: string

A stacktrace of what happened on the server when this error occured. Not always populated.

JSONPatch: object

A json patch operation, as defined here http://jsonpatch.com/

op: string , x ∈ { replace , add , remove , copy , move , test }
value: string

The new value to replace with

path: string

The path descripting which field to update (see the documentation on jsonpatch.com)

from: string

The path to the field you wish to move

NewObjectReference: object

A reference to the id of the newly created object

id: integer (int64)

Unique id of this particular test suite as an integer, use this if your parser can handle 64 bit values

idStr: string

Unique id of this particular test suite as a string use this if you have a sucky parser

Reference: object

A link to a resource

url: string

The url to the linked resource

rel: string

Name of the relationship this link models

State: string , x ∈ { Pending (default) , Running , Finished }

The state of this object in Muffin

Tags: string

A string indicating a grouping

TestSuite: object

id: integer (int64)

Unique id of this particular test suite as an integer, use this if your parser can handle 64 bit values

idStr: string

Unique id of this particular test suite as a string use this if you have a sucky parser

name: string

The name of the test suite

description: string

A longer description of that this test suite does

tags: object[]

A set of tags that indicate that can be used to group test suites

Tags
metadata: object

The metadata of the Test Suite, any patched values end up here. This is not versioned so you can never assume this contains particular fields.

references: object[]

The related resources, possible relationships are:

  • Runs
Reference

TestSuiteRun: object

id: integer (int64)

Unique id of this run of a test suite as an integer, use this if your parser can handle 64 bit values

idStr: string

Unique id of this run of a test suite as a string use this if you have a sucky parser

createdAt: string (datetime)

The time when this run was initially registered with Muffin.

startedAt: string (datetime)

The time when this run transitioned into the Running state.

endedAt: string (datetime)

The time when this run transitioned into the Finished state.

state: State
result: string

The result of the test suite as set when transitioning into the Finished state

metadata: object

The metadata of this TestSuiteRun, any patched values end up here. This is not versioned so you can never assume this contains particular fields.

references: object[]

  • TestSuite
